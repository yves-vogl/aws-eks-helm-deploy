# aws-eks-helm-deploy

A Bitbucket Pipelines Pipe that deploys Helm charts to AWS EKS — small, opinionated,
and supply-chain hardened. v2.x is the active line; v1.x is frozen.

## Where to next

- **[Migration v1 → v2](migration/v1-to-v2.md)** — read this FIRST if you're upgrading from `yvogl/aws-eks-helm-deploy:1.x`. Covers `INJECT_BITBUCKET_METADATA` flip, `NAMESPACE` correction, image-tag pinning policy, and the distribution change from Docker Hub to GHCR.
- **[Quickstart](quickstart/overview.md)** — drop a single pipe step into your `bitbucket-pipelines.yml`. The 60-second snippet also lives at the top of the [GitHub README](https://github.com/yves-vogl/aws-eks-helm-deploy).
- **[Variable reference](reference/variables.md)** — autogenerated from the v2 Pydantic Settings model on every commit.
- **[OIDC setup guide](guides/oidc-setup.md)** — IAM trust policy + Bitbucket OIDC step-by-step.
- **[ADRs](adr/index.md)** — every architectural decision shipped in v2.0, MADR 4.0 template.

## Supply chain

Every release image is signed with Cosign keyless (Sigstore / Fulcio / Rekor) and ships
SPDX + CycloneDX SBOMs and SLSA build provenance. See [SECURITY.md](https://github.com/yves-vogl/aws-eks-helm-deploy/blob/main/SECURITY.md) for the disclosure flow.
