    **New private helper `_run_helm_subcommand` (consolidates the subprocess call pattern):**

    ```python
    def _run_helm_subcommand(
        self,
        argv: list[str],
        *,
        env: dict[str, str],
        timeout: int,
        error_prefix: str,
    ) -> None:
        """Run a helm sub-command; raise ChartResolutionError on non-zero returncode.

        Unlike upgrade_install, chart-resolution sub-commands (repo add, repo update, pull)
        raise ChartResolutionError (exit_code=4), NOT HelmExecutionError (exit_code=5),
        because the failure surfaces in the CHART RESOLUTION step — before the helm
        upgrade --install action starts. Consumers reading the exit code can distinguish
        "couldn't find / pull the chart" (4) from "helm upgrade --install itself failed" (5).
        """
        try:
            result = subprocess.run(  # noqa: S603
                argv,
                capture_output=True,
                text=True,
                check=False,
                env=env,
                timeout=timeout,
            )
        except subprocess.TimeoutExpired as exc:
            partial_stderr = ""
            if exc.stderr is not None:
                raw = (
                    exc.stderr if isinstance(exc.stderr, str)
                    else exc.stderr.decode("utf-8", errors="replace")
                )
                partial_stderr = raw[-1024:]
            msg = f"{error_prefix} timed out after {exc.timeout}s"
            if partial_stderr:
                msg += f"; last stderr: {partial_stderr}"
            raise ChartResolutionError(msg) from exc

        if result.returncode != 0:
            truncated_stderr = _truncate_stderr(result.stderr)
            raise ChartResolutionError(
                f"{error_prefix} returned {result.returncode} — last stderr: {truncated_stderr}"
            )
        # Success — no return value needed; the subprocess wrote to env-scoped tempdir.
    ```

    ```python
    """RepoChart — Helm repository chart source.

    Requirements traceability:
      - CHART-02 (Phase 4): consumer sets CHART=repo://<name>/<chart> + REPO_URL + optional CHART_VERSION;
        this module runs `helm repo add` + `helm repo update` + `helm pull <name>/<chart>` into a tempdir
        and yields a ResolvedChart whose source_path points at the unpacked chart directory.

    Architecture:
      - All subprocess.run calls go through HelmClient (Phase 3 D1 invariant preserved). This module
        constructs a HelmClient internally (with kubeconfig_path=None — none needed for repo ops) and
        calls helm_client.repo_add/repo_update/pull_repo. The HelmClient routes each through
        subprocess.run from helm/client.py only.
      - Tempdir lifecycle mirrors kube/kubeconfig.py (CONTEXT D6): mkdtemp -> try/yield/finally(rmtree).
      - Env isolation: HELM_REPOSITORY_CONFIG + HELM_REPOSITORY_CACHE point at the tempdir so nothing
        leaks to ~/.config/helm/.
    """
    from __future__ import annotations

    import os
    import pathlib
    import shutil
    import tempfile
    from collections.abc import Iterator
    from contextlib import contextmanager

    from aws_eks_helm_deploy.chart.base import ResolvedChart
    from aws_eks_helm_deploy.chart.local import _parse_chart_yaml  # noqa: PLC0415
    from aws_eks_helm_deploy.errors import ChartResolutionError
    from aws_eks_helm_deploy.helm.client import HelmClient

    __all__: list[str] = ["RepoChart"]


    class RepoChart:
        """Resolves a chart from a Helm HTTP/HTTPS repository.

        Constructor stores args only — no I/O. The actual helm subprocess calls happen
        inside resolve() under the context-manager scope.
        """

        def __init__(
            self,
            name: str,
            chart: str,
            repo_url: str,
            version: str | None = None,
        ) -> None:
            self._name = name
            self._chart = chart
            self._repo_url = repo_url
            self._version = version

        @contextmanager
        def resolve(self) -> Iterator[ResolvedChart]:
            """Yield a ResolvedChart with source_path inside a tempdir.

            Tempdir is cleaned up on context exit even if the with-block raises
            (CONTEXT D6 + mirror of kube/kubeconfig.py finally/rmtree pattern).
            """
            tmpdir = pathlib.Path(tempfile.mkdtemp(prefix="aws-eks-helm-deploy-chart-"))
            try:
                unpack_dir = tmpdir / "unpacked"
                unpack_dir.mkdir()

                # Isolated env — nothing leaks to ~/.config/helm/
                env = os.environ.copy()
                env["HELM_REPOSITORY_CONFIG"] = str(tmpdir / "repositories.yaml")
                env["HELM_REPOSITORY_CACHE"] = str(tmpdir / "cache")

                # HelmClient with a placeholder kubeconfig_path — repo ops don't need it,
                # but the constructor requires SOMETHING. Pass a path inside the tempdir
                # that's never created/read.
                helm_client = HelmClient(kubeconfig_path=tmpdir / "unused-kubeconfig.yaml")

                # 1. helm repo add <name> <repo_url>
                helm_client.repo_add(self._name, self._repo_url, env)
                # 2. helm repo update <name>
                helm_client.repo_update(self._name, env)
                # 3. helm pull <name>/<chart> --destination tmp --untar --untar-dir tmp/unpacked
                repo_chart = f"{self._name}/{self._chart}"
                helm_client.pull_repo(repo_chart, tmpdir, unpack_dir, self._version, env)

                # 4. Single-subdir discovery (R7) — chart name from Chart.yaml,
                # NOT the tgz filename or repo ref path
                candidates = [p for p in unpack_dir.iterdir() if p.is_dir()]
                if len(candidates) != 1:
                    raise ChartResolutionError(
                        f"expected exactly one unpacked chart dir in {unpack_dir}, "
                        f"found {len(candidates)}"
                    )
                chart_dir = candidates[0]

                # 5. Parse Chart.yaml for name + version
                data = _parse_chart_yaml(chart_dir)
                yield ResolvedChart(
                    name=str(data.get("name", chart_dir.name)),
                    version=str(data.get("version", "")),
                    source_path=chart_dir,
                )
            finally:
                shutil.rmtree(tmpdir, ignore_errors=True)
    ```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| consumer `REPO_URL` → `helm repo add <name> <url>` | The repo URL is consumer-supplied; the pipe passes it as argv to helm (NOT as a shell string). `argv` is a list[str] — no shell injection risk. Helm validates the URL format; bad URLs surface as `ChartResolutionError`. |
| `helm repo update` cache → `<tmp>/cache/` | The helm cache is scoped to the per-invocation tempdir (HELM_REPOSITORY_CACHE). Cleaned up on context exit. No leakage to `~/.cache/helm/` or `~/.config/helm/`. |
| `helm pull` tarball + untar → `<tmp>/<chart>-<version>.tgz` + `<tmp>/unpacked/<chart>/` | The downloaded chart is in the tempdir. `shutil.rmtree(tmp, ignore_errors=True)` in the `finally` removes both the tarball AND the unpacked dir atomically. No file persists past the context-manager scope. |
| Single-subdir discovery (R7) → consumer chart name | The unpacked subdir name is whatever the chart's own `Chart.yaml` declares — NOT the repo path component. The discovery iterates the unpacked dir and asserts exactly one child. A malicious chart that contains multiple top-level dirs raises `ChartResolutionError` (the iteration assertion catches it). |

<deviations>
## Documented Deviations from CONTEXT / RESEARCH

### Deviation 1: New methods raise `ChartResolutionError` (NOT `HelmExecutionError`)

**Phase 3 invariant:** `HelmClient.upgrade_install` raises `HelmExecutionError(exit_code=5)` on non-zero returncode.

**Plan response (this plan):** the 3 new methods (`repo_add`, `repo_update`, `pull_repo`) raise `ChartResolutionError(exit_code=4)` instead. Rationale: chart-resolution failures (couldn't reach the repo, chart not found in index, version doesn't exist) are SEMANTICALLY different from helm-upgrade-itself failures. Consumers reading the exit code can distinguish "chart not found" (4) from "upgrade failed" (5).

**Action item:** Plan-Checker should NOT flag this as an inconsistency — the `_run_helm_subcommand` private helper accepts an `error_prefix` parameter and raises the appropriate typed error based on its caller. Documented in the helper's docstring.

### Deviation 2: HelmClient with placeholder kubeconfig_path

**HelmClient constructor REQUIRES a `kubeconfig_path: pathlib.Path` arg** (Phase 3). RepoChart's repo/pull operations don't need a kubeconfig (they don't touch the cluster).

**Plan response:** pass `kubeconfig_path=tmpdir / "unused-kubeconfig.yaml"` — a path that's never created, never read. The constructor stores it but the 3 new methods don't reference `self._kubeconfig_path` at all.

**Alternative considered:** make HelmClient's `kubeconfig_path` optional (`Optional[pathlib.Path] = None`). REJECTED: would break existing `upgrade_install`/`history` typing (those methods assume the path is set). The placeholder-path approach is a 1-line workaround vs. a typing refactor.

**Action item:** documented in the RepoChart code with a comment. Plan-Checker accepts.

### Deviation 3: Cross-module import of `_parse_chart_yaml` (private helper)

**Already documented in Plan 04-05 Deviation 2.** This plan honors the agreement: imports `from aws_eks_helm_deploy.chart.local import _parse_chart_yaml  # noqa: PLC0415`. The `# noqa: PLC0415` annotation suppresses the "import outside top-level" warning (the import is in fact at module top here, not function-local — but the underscore-prefix triggers PLC0415's other rule about importing-private-name).

**Action item:** Plan-Checker accepts.

### Deviation 4: Integration test uses `python -m http.server` instead of httpserver fixture

**Alternative considered:** use `pytest-httpserver` (would require adding a new dev dep).

**Plan response:** use the stdlib `python -m http.server` via subprocess.Popen. Rationale: NO new dev dependency; helm only needs an HTTP server that returns `index.yaml` + chart tarballs — `python -m http.server` does this trivially.

**Trade-off:** the fixture is slightly more verbose than httpserver would be. Acceptable cost.

**Action item:** none.
