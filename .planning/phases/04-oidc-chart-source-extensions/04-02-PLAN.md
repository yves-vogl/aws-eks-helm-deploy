## Documented Deviations from CONTEXT / RESEARCH

### Deviation 1: `registry_password: SecretStr | None` (NOT `str | None` as CONTEXT D6 sketches)

**CONTEXT D6 example imports a plain `str` for `registry_password`** in the OciChart constructor signature sketch (`registry_password: str | None`).

**Reality (per RESEARCH ┬¦R13 + ┬¦7.6 Note):** plain `str` allows `repr(settings)` and any structlog auto-binding to leak the password verbatim. The pipe's `AwsCredentials.__repr__` already masks `secret_access_key`; the Settings layer should do the same for the OCI registry password.

**Plan response:** type `registry_password` as `pydantic.SecretStr | None`. The single unwrap site in Plan 04-07's `OciChart._run_helm_registry_login` calls `.get_secret_value()` and passes the result to `subprocess.run(input=...)` (NOT argv). All other code paths see the masked SecretStr.

**Rationale:** ~2 lines of code; eliminates a documented leak vector (R4 / R13). Plan-Checker greps for `SecretStr` in `settings.py` AND confirms the only `.get_secret_value()` call site is in 04-07.

**Action item:** none ŌĆö the choice is encoded in the field type. Plan 04-07 inherits this decision; the constructor of `OciChart` accepts `SecretStr | None` (matching the settings type) and unwraps internally.
