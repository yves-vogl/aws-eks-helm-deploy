<objective>
Ship the cosign-verify PR-gate per CONTEXT D7 — a defensive workflow that catches the three failure modes of Cosign keyless coupling that Plan 06-04 alone cannot detect:

1. Missing `id-token: write` permission re-introduced by a refactor of release.yml (verify would fail because the next release would produce no signature).
2. Rekor unavailability that wasn't caught at release time (the signature exists in GHCR but doesn't validate against the transparency log).
3. An accidental re-tag of `:latest` without re-signing (someone manually pushes to GHCR; the digest changes but cosign verify fails).

Purpose: SEC-01 says "every release image is signed with Cosign keyless ... offline `--bundle`". Plan 06-04 produces the signature. THIS plan ensures the signature stays verifiable — every PR runs `cosign verify` against the current `:latest` and `:2` tags. A signature regression blocks merge.

Output:
- `.github/workflows/cosign-verify.yml` — minimal-permissions, pull_request-triggered, bootstrap-graceful workflow.
- `tests/structural/test_cosign_verify_yml.py` — structural assertions.
</objective>

**Full workflow content:**

```yaml
# Phase 6 / SEC-01 / CONTEXT D7 — Cosign verify PR-gate.
# Runs on every PR against main; verifies that the most recent published GHCR image
# has a valid Cosign keyless signature (Pitfall #4 mitigation) AND that both SBOM
# attestations (SPDX + CycloneDX) are present and verifiable. Failure blocks merge.
#
# Pitfall #1: This workflow is `pull_request`-triggered. It MUST NOT request
# id-token:write — verification is a READ operation and PR-triggered jobs must
# never have OIDC token elevation. Cosign verify works fine without id-token.
#
# Open Question 3 / bootstrap path: until the first release ships, :latest does not
# exist on GHCR — the workflow detects this and exits 0 with a notice.
name: cosign-verify

on:
  pull_request:
    branches: [main]
  workflow_dispatch:

permissions:
  contents: read

concurrency:
  group: ${{ github.workflow }}-${{ github.ref }}
  cancel-in-progress: true

env:
  REGISTRY: ghcr.io
  IMAGE_NAME: yves-vogl/aws-eks-helm-deploy
  CERT_IDENTITY_REGEXP: '^https://github.com/yves-vogl/aws-eks-helm-deploy/'
  CERT_OIDC_ISSUER: 'https://token.actions.githubusercontent.com'

jobs:
  verify:
    name: cosign verify (latest GHCR release)
    runs-on: ubuntu-24.04
    timeout-minutes: 5
    steps:
      - name: Checkout
        uses: actions/checkout@11bd71901bbe5b1630ceea73d27597364c9af683 # v4.2.2

      - name: Install cosign
        uses: sigstore/cosign-installer@6f9f17788090df1f26f669e9d70d6ae9567deba6 # v4.1.2
        with:
          cosign-release: "v2.6.3"

      - name: Probe :latest existence (bootstrap-graceful)
        id: probe
        # Use anonymous manifest HEAD via curl — no docker login needed for a public
        # registry probe. If the manifest doesn't exist (HTTP 404), we set exists=false
        # and the subsequent verify steps skip via `if:`.
        run: |
          set -uo pipefail
          IMAGE="${REGISTRY}/${IMAGE_NAME}"
          # GHCR requires Bearer-token-of-empty-string for public-registry anon access.
          MANIFEST_URL="https://${REGISTRY}/v2/${IMAGE_NAME}/manifests/latest"
          STATUS=$(curl -sSL -o /dev/null -w '%{http_code}' \
            -H 'Accept: application/vnd.oci.image.index.v1+json' \
            -H 'Accept: application/vnd.docker.distribution.manifest.list.v2+json' \
            "${MANIFEST_URL}" || echo "000")
          if [[ "$STATUS" == "200" ]]; then
            echo "exists=true" >> "$GITHUB_OUTPUT"
            echo "::notice::Latest image manifest found at ${MANIFEST_URL}"
          else
            echo "exists=false" >> "$GITHUB_OUTPUT"
            echo "::notice::Latest image manifest not found (HTTP ${STATUS}) — assuming pre-first-release bootstrap; skipping verify."
          fi

      - name: Verify image signature (Cosign keyless)
        if: steps.probe.outputs.exists == 'true'
        env:
          IMAGE_REF: ${{ env.REGISTRY }}/${{ env.IMAGE_NAME }}:latest
        run: |
          set -euo pipefail
          cosign verify \
            --certificate-identity-regexp "${CERT_IDENTITY_REGEXP}" \
            --certificate-oidc-issuer "${CERT_OIDC_ISSUER}" \
            "${IMAGE_REF}"

      - name: Verify SPDX SBOM attestation
        if: steps.probe.outputs.exists == 'true'
        env:
          IMAGE_REF: ${{ env.REGISTRY }}/${{ env.IMAGE_NAME }}:latest
        run: |
          set -euo pipefail
          cosign verify-attestation \
            --type spdxjson \
            --certificate-identity-regexp "${CERT_IDENTITY_REGEXP}" \
            --certificate-oidc-issuer "${CERT_OIDC_ISSUER}" \
            "${IMAGE_REF}" > /dev/null

      - name: Verify CycloneDX SBOM attestation
        if: steps.probe.outputs.exists == 'true'
        env:
          IMAGE_REF: ${{ env.REGISTRY }}/${{ env.IMAGE_NAME }}:latest
        run: |
          set -euo pipefail
          cosign verify-attestation \
            --type cyclonedx \
            --certificate-identity-regexp "${CERT_IDENTITY_REGEXP}" \
            --certificate-oidc-issuer "${CERT_OIDC_ISSUER}" \
            "${IMAGE_REF}" > /dev/null

      - name: Verify :2 rolling-major tag (defense in depth, post-first-release only)
        if: steps.probe.outputs.exists == 'true'
        env:
          IMAGE_REF: ${{ env.REGISTRY }}/${{ env.IMAGE_NAME }}:2
        run: |
          set -uo pipefail
          # :2 may not exist if the first release is a non-v2 prerelease — graceful.
          if cosign verify \
            --certificate-identity-regexp "${CERT_IDENTITY_REGEXP}" \
            --certificate-oidc-issuer "${CERT_OIDC_ISSUER}" \
            "${IMAGE_REF}" > /dev/null 2>&1; then
            echo "::notice::Verified signature on ${IMAGE_REF}"
          else
            echo "::warning::Could not verify ${IMAGE_REF} — may not exist yet (pre-v2.0.0 release)"
          fi
```

**Notes on each step:**

- **Trigger** — `pull_request: branches: [main]` per D7. NO `pull_request_target` (would be a Pitfall #1 violation since the verify needs no privileged access). `workflow_dispatch` enables manual smoke-runs.
- **Permissions** — `contents: read` ONLY at workflow level; NO `id-token: write` (Pitfall #1 invariant — cosign verify works without it). Cosign verify reads from the public Rekor log + the public Fulcio cert; no OIDC token needed in the verifier.
- **`env`** block — single source-of-truth for the cert-identity regexp + issuer. The regexp `^https://github.com/yves-vogl/aws-eks-helm-deploy/` matches the cert subject that Fulcio issues for any workflow in this repo (e.g., `https://github.com/yves-vogl/aws-eks-helm-deploy/.github/workflows/release.yml@refs/tags/v2.0.0`). Tightening to a specific workflow path is brittle (workflow paths change) — the repo-scoped regexp is the right balance.
- **`probe` step** — uses curl + a HEAD-equivalent request against the GHCR Distribution Spec endpoint. If the manifest exists, sets `exists=true`. If not (HTTP 404 — the pre-first-release bootstrap case), sets `exists=false` and the subsequent steps skip via `if:`.
- **`cosign verify` step** — verifies the image signature against the certificate identity (this repo) and OIDC issuer (GitHub Actions). Uses `--certificate-identity-regexp` (NOT `--certificate-identity`) because the exact path includes a workflow ref that varies per release.
- **2x `cosign verify-attestation`** — defense-in-depth for SEC-02: BOTH SBOM attestations must remain verifiable. A regression where only one SBOM is attested would slip through Plan 06-04's structural test but is caught at PR time by this gate.
- **`:2` verify step** — also graceful (the warning vs. error split). The `:2` tag only exists after the first v2.x release; before that, this step warns but doesn't fail.
- **Concurrency `cancel-in-progress: true`** — PR-triggered workflow; superseded runs are cancelled (matches ci.yml from Plan 06-01).
- **Job timeout: 5 minutes** — cosign verify against GHCR is fast (< 30s in practice); 5min is the safety budget.

**Forbidden in this file:**

- NO `id-token: write` ANYWHERE (Pitfall #1 — workflow- or job-level).
- NO `pull_request_target` (security antipattern — Pitfall #1).
- NO `${{ secrets.* }}` in `if:` conditions (the only `if:` is `steps.probe.outputs.exists == 'true'` — pure step-output reference, not a secret).
- NO unpinned action references.
- NO use of `docker login` to a private registry — cosign verify reads from PUBLIC GHCR, no auth needed.

| cosign-verify.yml → Sigstore Rekor + Fulcio | Read-only; cosign verify walks the public transparency log. |

| T-06-05-04 | Information Disclosure | cosign verify exposes cert chain in logs | accept | Cert chain is PUBLIC (Sigstore transparency model). Not a credential. |
