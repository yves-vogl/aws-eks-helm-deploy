# OpenSSF Scorecard Exceptions

This file documents OpenSSF Scorecard check results that are intentionally allowed
to score below the target (>= 8/10 at v2.0 tag-cut per SEC-10), with time-bound
rationale. Each entry must be reviewed before `review_date`. CI enforces stale
detection -- a past `review_date` fails the build.

## Grammar (YAML frontmatter)

```yaml
exceptions:
  - check: <Scorecard check name, e.g. Token-Permissions>
    reason: "<short prose explaining why this sub-check is allowed to fail>"
    review_date: YYYY-MM-DD     # must be in the future AND within 180 days of today
    owner: <github-handle>
```
